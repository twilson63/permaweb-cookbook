# ArFS Protocol: A Decentralized File System on Arweave

Arweave File System、略して「ArFS」は、一般的なファイルシステム操作を模倣し、[Arweave](https://ardrive.io/what-is-arweave/) の恒久的かつ不変なデータ保存である blockweave（ブロックウィーブ）上におけるデータ階層に対して一部の可変性を提供するために設計された、データモデリング、保存、取得のプロトコルです。

Arweave の恒久的・不変かつ公開の性質により、従来のファイルシステム操作（権限設定、ファイル/フォルダの名前変更や移動、ファイルの更新など）は、オンチェーンのデータモデルを単純に更新するだけでは実現できません。

ArFS はこれを回避するために、プライバシーおよび暗号化パターンを実装し、[Arweave Transaction headers](https://docs.arweave.org/developers/server/http-api#transaction-format) 内のタグを利用した追記専用（append-only）のトランザクションデータモデルを定義します。

## 主な特徴

### ファイル構造

ArFS は階層構造を用いてファイルとフォルダを整理します。ファイルは Arweave ブロックチェーン上の個々のトランザクションとして保存され、フォルダはこれらのファイルトランザクションを参照するメタデータとして表現されます。

### メタデータ

各ファイルおよびフォルダには、名前、種類、サイズ、最終更新タイムスタンプなどの関連メタデータがあります。ArFS は Arweave のタグシステムを利用して、このメタデータを標準化された形式で保存し、容易なクエリと整理を可能にします。

### ファイル権限

ArFS はパブリックおよびプライベートのファイル権限をサポートします。パブリックファイルはネットワーク上の誰でもアクセス可能であり、プライベートファイルは所有者の秘密鍵を用いて暗号化され、所有者だけが復号して内容にアクセスできるようにします。

### ファイルのバージョン管理

ArFS はファイルのバージョン管理をサポートし、ユーザーがファイルの複数バージョンを保存し、任意の過去バージョンにアクセスできるようにします。これはメタデータタグを用いて新しいファイルトランザクションを前のバージョンにリンクすることで実現されます。

### データ重複排除

保存の冗長性とコストを最小化するために、ArFS はデータ重複排除の技術を採用しています。ユーザーがネットワーク上に既に存在するファイルを保存しようとした場合、プロトコルは重複したコピーを保存するのではなく、既存のファイルへの新しい参照を作成します。

### 検索と発見

ArFS はファイル名、種類、タグなどのメタデータに基づいてファイルを検索・発見できるようにします。これは Arweave ブロックチェーン内に保存されたメタデータをインデックス化することで実現されます。

### 相互運用性

ArFS は Arweave ネットワーク上で構築された他の分散型アプリケーションやサービスと相互運用可能となるよう設計されています。これにより、異なるアプリケーションやユーザー間でのシームレスな統合と協働が可能になります。

## はじめに

ArFS を使い始めるには、Arweave エコシステムに慣れ、ストレージコストをカバーするための AR トークンを取得し、ArFS プロトコルとやり取りするための互換クライアントまたはライブラリを選択する必要があります。
