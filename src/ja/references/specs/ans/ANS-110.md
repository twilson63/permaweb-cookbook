# [ANS-110: 資産の検出性](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-110.md)

ステータス: 草案

著者: Tom Wilson (tom@hyper.io), Sam Williams (sam@arweave.org), Abhav Kedia (abhav@arweave.org)

## 概要

本書は、permaweb 上の資産を検出し一貫して表示することを可能にするデータプロトコルを定義します。

## 動機

permaweb はメディア、コンテンツ、関数、アプリケーションなど多様なデータの集合体です。資産を識別するための標準プロトコルは、ダッシュボード、取引所、上位レベルの permaweb サービスに対する検出可能性を可能にします。トランザクション向けに拡張可能なデータプロトコルを提供することで、作成者や発行者は permaweb 全体でのコンテンツ検出機能を活用でき、さまざまなアプリケーションやコンテキストで簡単に表示されるようになります。

例えば、本プロトコルはメディア資産をユーザーフレンドリーに表示するマーケットプレイスや取引所を可能にします。これらの識別子を用いて資産トランザクションを拡張することで、作成者はマーケットプレイスや取引所が資産の詳細な説明を行うために使用できる明確かつ合成可能な識別子群を提供できます。別の例として、特定種類の資産をインデックス化したい検索エンジンサービスが挙げられます。

## 仕様

### トランザクション形式

Asset Discoverability を利用するには、作成者または発行者が以下のタグを指定して Arweave トランザクションを送信または投稿できます。

| タグ名      | 任意? | タグ値                                                                                                                                                                                        |
| ----------- | ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Title       | False | コンテンツを識別するために使用される最大 150 文字のタイトル。資産の簡潔で目を引く説明を提供できます。                                                                                         |
| Type\*      | False | 資産のタイプ。次のいずれか一つ以上: `meme`, `image`, `video`, `podcast`, `blog-post`, `social-post`, `music`, `token`, `web-page`, `profile`                                                  |
| Topic\*     | True  | 特定のトピックによって与えられたタイプの資産を検索するために使用できる、0 個以上のトピック。例えば、タイプが `meme` の資産は次のような 2 つのトピックを持つ場合があります: `Funny`, `Sports`. |
| Description | True  | 資産をさらに説明するための、最大 300 文字の詳細な説明                                                                                                                                         |

### 利用方法

これらのタグの主な目的は、コンテンツ開発者が GraphQL を利用してアプリケーションで使用する特定のタイプやトピックの資産トランザクションを検索できるようにすることです。

```javascript
query {
  transactions(
      first: 100,
        tags: [
          { name: "Type", values: ["meme", "blog-post"] },
          { name: "Topic:Funny", values: ["Funny"] },
          { name: "Topic:Jokes", values: ["Jokes"] }
        ]) {
      edges {
          node {
              id
                owner {
                  address
                }
                tags {
                  name
                    value
                }
        }}
    }
}
```

この GraphQL クエリは `Type` および `Topic` タグに基づいてフィルタリングし、資産を集約リスト表示用に絞り込みます。
