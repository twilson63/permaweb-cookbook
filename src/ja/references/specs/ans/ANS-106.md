# [ANS-106: Do-Not-Store リクエスト](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-106.md)

ステータス: ドラフト-1

著者: Abhav Kedia <abhav@arweave.org>, Sam Williams <sam@arweave.org>

## 概要

本書は、Arweave ネットワークの利用者が特定の種類のデータをマイナーに保存しないよう要求するために使用できるトランザクション形式について説明します。非保存を要求するには、対象データのトランザクション ID と、本仕様に従ったタグを付与してトランザクションをアップロードしてください。この要求を受け取ったネットワーク上のノードは、独自にその要求を受け入れるかどうかを判断します。

## 動機

Arweave の永続ストレージプロトコルは、メディアや文書の真に永続的なデジタル保存を可能にします。

人物や企業がネットワークから特定データを削除したいと望む理由は様々です。これにはプライバシー、政府規制、著作権侵害などが含まれます。本標準に準拠したトランザクションをアップロードすることで、ネットワーク利用者はその要求が関連するストレージノード運用者全員にブロードキャストされることを確実にできます。

## 仕様

### トランザクション形式

Do-Not-Store リクエストトランザクションは、次のタグを含むトランザクションでなければなりません。

| タグ名       | _必須?_ | タグ値                                                                                                 |
| ------------ | ------- | ------------------------------------------------------------------------------------------------------ |
| App-Name     | False   | `Do-Not-Store`                                                                                         |
| Do-Not-Store | False   | Arweave `Txn ID` of the data in question                                                               |
| Category     | True    | `Category-Tag`                                                                                         |
| Geography    | True    | If requesting removal in particular countries, include the 2 letter `Country-Code` (Alpha-2, ISO 3166) |
| Content-Type | True    | A guide to the type of content included in the body of the transaction, in order to aid rendering.     |

トランザクションの本文には、データを保存すべきでない理由を説明するテキスト記述を含める必要があります。複数の Do-Not-Store タグを追加して、複数のデータ項目の非保存を同時に要求することができます。

### カテゴリタグ

カテゴリタグは、削除理由を簡潔に指定するための省略表記です。一般的なタグには `Private`, `Regulation`, `Copyright` などがあります。必要に応じてカスタムのカテゴリタグを使用できます。カスタムタグは 50 文字を超えてはなりません。

## 今後の作業

データやそれに関連するタグのさまざまな属性に基づいて削除を要求する必要が生じる可能性があります。本標準は、トランザクションのリストや関連するフィルタリング基準を許可するよう拡張される可能性があります。
