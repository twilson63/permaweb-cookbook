# [ANS-109: Vouch-For (身元の主張)](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-109.md)

ステータス: 草案（バージョン `0.1`）

著者: Abhav Kedia (abhav@arweave.org), Sam Williams (sam@arweave.org), Tom Wilson (tom@hyper.io)

## 概要

本書は、アドレスが permaweb 上の他のアドレスの身元を保証する（vouch for）ためのトランザクション形式を定義します。

## 動機

Sybil 耐性は permaweb 上のほとんどのアプリケーションにとって必要不可欠な要素です。他のアドレスの身元を「保証」できるトランザクション形式は、人間および自動化された検証者がアドレスが人間であることを確認することを可能にします。その他の全てのアプリケーションは、この情報を身元確認の基礎的な手段として利用でき、さらにその上に様々な保護策を構築できます。

このようなシステムの上に構築され得る抽象化や保護策の一例として、「VouchDAO」が挙げられます — これは、ある時点でどの人間の検証者（Verifier）や検証サービス（Verification Services）を信頼できるとコミュニティが指定する仕組みです。

## 仕様

### トランザクション形式

検証者は、以下のタグを含むトランザクションを送信することで、`Vouch-For` 標準を使ってある `address` の身元を主張できます。

| タグ名              | _オプション?_ | タグ値                                                                                                |
| ------------------- | ------------- | ----------------------------------------------------------------------------------------------------- |
| App-Name            | False         | `Vouch`                                                                                               |
| Vouch-For           | False         | Arweave `address` that is being vouched for in this transaction                                       |
| App-Version         | True          | `0.1`                                                                                                 |
| Verification-Method | True          | Method of verification of identity for the person. Example - `Twitter`/`In-Person`/`Gmail`/`Facebook` |
| User-Identifier     | True          | An identifier for the user based on the Verification Method. Example - `abhav@arweave.org`            |

### 利用方法

本標準の利用者は、arweave ネットワーク上で特定の `address` の身元を保証する `Vouch-For` 標準のトランザクションを対象に graphql クエリを実行できます。例えば、

```javascript
query {
  transactions(
    tags:{name:"Vouch-For", values:["0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g"]}
  ) {
    edges {
      node {
        id
        tags {
          name
          value
        }
      }
    }
  }
}
```

このクエリはアドレス `0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g` への全ての vouch を返します。上記で概説した「VouchDAO」の実装などによる追加フィルタは、検証者として指定された特定のオーナーでフィルタリングすることで適用できます。
