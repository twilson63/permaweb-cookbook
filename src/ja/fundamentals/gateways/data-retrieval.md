# Arweave からのデータ取得

トランザクション ID (txid) または ArNS 名を取得したら、Arweave ネットワークから関連データを取得する方法はいくつかあります。本ガイドでは、開発者が利用できる主なアプローチを解説します。

## 概要

Arweave に格納されたデータを取得する必要がある場合、ユースケースに応じて複数の選択肢があります:

- **HTTP API**: ゲートウェイエンドポイントへの直接 HTTP リクエスト
- **Arweave.js**: プログラムからのアクセス用 JavaScript/TypeScript SDK
- **ARIO Wayfinder**: インテリジェントなルーティングと検証を行うプロトコル

## 利用可能な方法

### [HTTP API](./http-api.md)

標準的な HTTP リクエストを Arweave ゲートウェイに送る最も簡単な方法です。基本的なデータ取得や依存関係を最小限にしたい場合に最適です。

**適しているケース:**

- 単純なデータ取得
- サーバーサイドアプリケーション
- 追加パッケージを避けたい場合

### [Arweave.js](./arweave-js.md)

Arweave ネットワークに対する包括的なインターフェースを提供する公式の JavaScript/TypeScript SDK です。

**適しているケース:**

- JavaScript/TypeScript アプリケーション
- 複雑な Arweave 操作
- トランザクションのメタデータやデータが必要な場合

### [ARIO Wayfinder](./wayfinder.md)

分散型で暗号学的に検証されたアクセスと、インテリジェントなゲートウェイルーティングを提供するプロトコルです。

**適しているケース:**

- 信頼性を要する本番用途のアプリケーション
- 自動ゲートウェイ選択が必要な場合
- データ検証を必要とするアプリケーション

## 簡易比較

| Method     | Complexity | Dependencies            | Features                             |
| ---------- | ---------- | ----------------------- | ------------------------------------ |
| HTTP API   | 低         | なし                    | 基本的なデータ取得                   |
| Arweave.js | 中         | `arweave` package       | トランザクション全体へのアクセス     |
| Wayfinder  | 中         | `@ar.io/wayfinder-core` | インテリジェントなルーティング、検証 |

## はじめに

ニーズに最も適した方法を選んでください:

1. **HTTP API から始める** — 最もシンプルなアプローチを取りたい場合
2. **Arweave.js を使用する** — JavaScript/TypeScript アプリケーションを構築する場合
3. **Wayfinder を検討する** — 本番レベルの信頼性と検証が必要な場合

各手法のページにはインストール手順、基本的な使用例、および詳細なドキュメントへのリンクが含まれています。
