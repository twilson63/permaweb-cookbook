## 커밋하기

이 리포지토리에서는 [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/)를 사용합니다.

기여를 위한 일반적인 흐름:

1. GitHub에서 리포지토리를 Fork
2. 프로젝트를 자신의 머신으로 Clone
3. 자신의 브랜치에 변경사항을 Commit
4. 작업을 자신의 Fork에 Push
5. 변경사항을 검토할 수 있도록 Pull request 제출

**참고**: Pull request를 만들기 전에 반드시 "upstream"으로부터 최신을 병합하세요!

## 작성 스타일

다음은 일부 기여자들이 제안하는 문체와 스타일에 대한 권장사항입니다.

::: tip
작성하면서 각 섹션에 적합한 어조를 파악하고 있습니다.
CoreConcepts는 교과서식으로, 중립적이고 객관적인 어조가 적절합니다. "This is how Arweave works"처럼요.
Guides의 경우에는 더 개인적인 목소리를 사용하는 것이 괜찮다고 생각합니다. 독자를 "you"로 지칭하고 "다음으로 살펴보겠습니다..."처럼 협력적인 어조로 말하세요.
이건 개인적인 취향일 수 있지만, 일반적으로 긴 형식의 가이드를 따라갈 때 이 어조가 더 친절하고 접근성이 높다고 느껴집니다.
실제로 다른 생태계에서 인기 있는 튜토리얼들도 대부분 이러한 어조로 작성되어 있습니다.
Resources는 CoreConcepts와 같은 어조를 공유하되, 간결함을 선호합니다.

dmac
:::

::: tip
개념적이고 참조용인 데이터는 보다 냉정한 과학적 어조를 가져야 하며, 가이드는 지지적이거나 심지어 유머러스한 어조를 사용할 수 있습니다. 긴 형식의 콘텐츠는 독자가 멍해지지 않도록 끌어들이는 요소가 필요합니다.

Arch_Druid
:::
