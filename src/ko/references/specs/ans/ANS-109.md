# [ANS-109: Vouch-For (신원 주장)](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-109.md)

상태: 초안 (버전 `0.1`)

저자: Abhav Kedia (abhav@arweave.org), Sam Williams (sam@arweave.org), Tom Wilson (tom@hyper.io)

## 개요

이 문서는 주소들이 퍼마웹(permaweb) 상에서 다른 주소의 신원을 보증(vouch)할 수 있도록 하는 트랜잭션 형식을 규정합니다.

## 동기

Sybil 저항성은 퍼마웹 상의 대부분 애플리케이션에 필요한 구성 요소입니다. 주소가 다른 주소의 신원을 `vouch`할 수 있는 트랜잭션 형식은 사람 및 프로그램 방식의 검증자(Verifier)들이 주소의 인간성을 확인할 수 있게 합니다. 모든 다른 애플리케이션은 이후 이러한 정보를 신원 확인의 원시 수단(primitive)으로 활용할 수 있으며, 그 위에 다양한 안전장치를 구축할 수 있습니다.

이러한 시스템 위에 구축될 수 있는 추상화 및 안전장치의 한 예로는 "VouchDAO"가 있을 수 있습니다. 이는 특정 시점에 신뢰할 수 있다고 간주되는 사람 기반의 "검증자(Verifiers)" 또는 "검증 서비스"를 커뮤니티가 명시하는 형태입니다.

## 명세

### 트랜잭션 형식

검증자(Verifier)는 다음 태그들을 포함한 트랜잭션을 전송함으로써 `Vouch-For` 표준을 사용해 특정 `address`의 신원을 주장할 수 있습니다.

| 태그 이름           | _선택적?_ | 태그 값                                                                |
| ------------------- | --------- | ---------------------------------------------------------------------- |
| App-Name            | False     | `Vouch`                                                                |
| Vouch-For           | False     | Arweave `address` that is being vouched for in this transaction        |
| App-Version         | True      | `0.1`                                                                  |
| Verification-Method | True      | 신원 확인 방법. 예시 - `Twitter`/`In-Person`/`Gmail`/`Facebook`        |
| User-Identifier     | True      | Verification Method에 기반한 사용자 식별자. 예시 - `abhav@arweave.org` |

### 사용법

이 표준의 사용자는 특정 `address`에 대해 보증(vouch)하는 `Vouch-For` 표준 트랜잭션들을 아르위브 네트워크에서 graphql 쿼리로 조회할 수 있습니다. 예를 들어,

```javascript
query {
  transactions(
    tags:{name:"Vouch-For", values:["0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g"]}
  ) {
    edges {
      node {
        id
        tags {
          name
          value
        }
      }
    }
  }
}
```

이 쿼리는 주소 `0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g`에 대한 모든 vouch들을 반환합니다. 위에서 설명한 "VouchDAO" 구현과 같이 추가 필터(예: 검증자로 지정된 특정 소유자들로 필터링)를 적용할 수 있습니다.
