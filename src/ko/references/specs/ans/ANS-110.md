# [ANS-110: 자산 검색성](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-110.md)

상태: 초안

저자: Tom Wilson (tom@hyper.io), Sam Williams (sam@arweave.org), Abhav Kedia (abhav@arweave.org)

## 요약

이 문서는 자산을 영구웹(permaweb)에서 검색하고 일관되게 표시할 수 있도록 하는 데이터 프로토콜을 규정합니다.

## 동기

퍼마웹은 미디어, 콘텐츠, 함수 및 애플리케이션과 같은 다양한 종류의 데이터로 풍부하게 구성되어 있습니다. 자산을 식별하기 위한 표준 프로토콜은 대시보드, 거래소 및 상위 수준의 퍼마웹 서비스에 대한 검색 가능성을 가능하게 합니다. 트랜잭션에 대해 확장 가능한 데이터 프로토콜을 제공함으로써, 제작자와 게시자는 퍼마웹 전반의 콘텐츠 검색 기능을 활용할 수 있으며 — 이를 통해 다양한 애플리케이션 및 문맥에서 자산이 쉽게 노출되도록 할 수 있습니다.

예를 들어, 이 프로토콜은 사용자가 미디어 자산을 거래하는 마켓플레이스나 거래소에서 사용자 친화적으로 렌더링할 수 있도록 합니다. 자산 트랜잭션을 이러한 식별자로 확장하면 제작자는 마켓플레이스나 거래소가 자산에 대한 자세한 설명을 제공하는 데 사용할 수 있는 명확하고 조합 가능한 식별자 집합을 제공합니다. 또 다른 예로는 특정 유형의 자산을 색인화하려는 검색 엔진 서비스가 있을 수 있습니다.

## 명세

### 트랜잭션 형식

자산 검색성을 활용하려면 제작자 또는 게시자는 다음 태그를 지정하여 Arweave 트랜잭션을 전송하거나 게시할 수 있습니다.

| 태그 이름   | _선택사항?_ | 태그 값                                                                                                                                                                         |
| ----------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Title       | False       | 콘텐츠를 식별하는 데 사용되는 최대 150자. 이 제목은 자산을 빠르게 눈에 띄게 설명할 수 있습니다.                                                                                 |
| Type\*      | False       | 자산의 유형. 하나 이상: `meme`, `image`, `video`, `podcast`, `blog-post`, `social-post`, `music`, `token`, `web-page`, `profile`                                                |
| Topic\*     | True        | 특정 유형의 자산을 특정 주제로 찾아내는 데 사용할 수 있는 0개 이상(없음 또는 여러 개)의 주제. 예: 유형이 meme인 자산은 다음 두 가지 주제를 가질 수 있습니다: `Funny`, `Sports`. |
| Description | True        | 자산을 추가로 설명할 수 있는 최대 300자의 긴 설명                                                                                                                               |

### 사용

이 태그들의 주요 목적은 콘텐츠 개발자가 GraphQL을 활용하여 애플리케이션에서 사용하기 위해 특정 유형 또는 주제의 자산 트랜잭션을 찾을 수 있게 하는 것입니다.

```javascript
query {
  transactions(
      first: 100,
        tags: [
          { name: "Type", values: ["meme", "blog-post"] },
          { name: "Topic:Funny", values: ["Funny"] },
          { name: "Topic:Jokes", values: ["Jokes"] }
        ]) {
      edges {
          node {
              id
                owner {
                  address
                }
                tags {
                  name
                    value
                }
        }}
    }
}
```

이 GraphQL 쿼리는 `Type` 및 `Topic` 태그를 기반으로 필터링하여 집계 목록 표시를 위한 자산을 필터링합니다.
