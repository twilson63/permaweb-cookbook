# [ANS-106: Do-Not-Store 요청](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-106.md)

상태: Draft-1

저자: Abhav Kedia <abhav@arweave.org>, Sam Williams <sam@arweave.org>

## 개요

이 문서는 Arweave 네트워크 사용자가 특정 유형의 데이터를 저장하지 말도록 마이너들에게 요청할 때 사용할 수 있는 트랜잭션 형식을 설명합니다. 비저장을 요청하려면 대상 데이터의 트랜잭션 ID와 여기 명시된 태그들을 포함한 트랜잭션을 업로드하십시오. 이 데이터를 수신한 네트워크의 노드들은 요청을 수락할지 여부를 독립적으로 결정합니다.

## 동기

Arweave의 영구 저장 프로토콜은 미디어와 문서의 진정한 영구적 디지털 저장을 가능하게 합니다.

개인 정보 보호, 정부 규제, 저작권 위반 등 여러 이유로 개인이나 단체가 네트워크에서 특정 데이터를 제거하길 원할 수 있습니다. 이 표준을 준수하는 트랜잭션을 업로드함으로써 네트워크 사용자는 관련 저장 노드 운영자들에게 자신의 요청이 브로드캐스트되도록 보장할 수 있습니다.

## 명세

### 트랜잭션 형식

Do-Not-Store 요청 트랜잭션은 다음 태그들을 포함한 트랜잭션이어야 합니다:

| 태그 이름    | _선택적?_ | 태그 값                                                                                |
| ------------ | --------- | -------------------------------------------------------------------------------------- |
| App-Name     | False     | `Do-Not-Store`                                                                         |
| Do-Not-Store | False     | Arweave `Txn ID` of the data in question                                               |
| Category     | True      | `Category-Tag`                                                                         |
| Geography    | True      | 특정 국가에서의 제거를 요청하는 경우 2글자의 `Country-Code` (Alpha-2, ISO 3166)를 포함 |
| Content-Type | True      | 트랜잭션 본문에 포함된 콘텐츠 유형을 안내하여 렌더링에 도움을 줌.                      |

트랜잭션 본문은 데이터가 저장되어서는 안 되는 이유를 설명하는 텍스트 설명을 포함해야 합니다. 여러 개의 Do-Not-Store 태그를 추가하여 한 번에 여러 데이터 항목의 비저장을 요청할 수 있습니다.

### 카테고리 태그

카테고리 태그는 제거 사유를 지정하는 간단한 표기 방법입니다. 일반적인 태그에는 `Private`, `Regulation`, `Copyright` 등이 있습니다. 필요에 따라 사용자 지정 카테고리 태그를 사용할 수 있습니다. 사용자 지정 태그는 50자를 초과해서는 안 됩니다.

## 향후 작업

데이터 또는 해당 데이터에 연결된 태그의 다양한 속성에 기반해 제거를 요청해야 할 필요가 있을 수 있습니다. 이 표준은 트랜잭션 목록 및/또는 연관된 필터링 기준을 허용하도록 확장될 수 있습니다.
