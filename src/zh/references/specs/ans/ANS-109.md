# [ANS-109：Vouch-For（身份断言）](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-109.md)

状态：草案（版本 `0.1`）

作者：Abhav Kedia (abhav@arweave.org), Sam Williams (sam@arweave.org), Tom Wilson (tom@hyper.io)

## 摘要

本文件定义一种交易格式，允许地址在 permaweb 上为其他地址的身份进行担保。

## 动机

Sybil 抵抗性是 permaweb 上大多数应用所需的必要组件。允许地址为其他地址进行「担保（vouch）」的交易格式，可使人工与程序化的验证者（Verifiers）确认地址的真人属性。其他应用即可将此信息作为身份验证的原始构件，并在其上建立各种保护机制。

举例来说，基于此系统之上的抽象与保护机制之一，可能是「VouchDAO」——一个社区，用以在特定时间点指定他们认为值得信赖的人类「Verifier」或「Verification Services」。

## 规范

### 交易格式

验证者可以通过提交包含下列标签的交易，使用 `Vouch-For` 标准来断言某个 `address` 的身份。

| 標籤名稱            | _是否为可选?_ | 標籤值                                                                  |
| ------------------- | ------------: | ----------------------------------------------------------------------- |
| App-Name            |            否 | `Vouch`                                                                 |
| Vouch-For           |            否 | Arweave `address` 在此交易中被担保的目标地址                            |
| App-Version         |            是 | `0.1`                                                                   |
| Verification-Method |            是 | 用于验证该人身份的方法。例如 - `Twitter`/`In-Person`/`Gmail`/`Facebook` |
| User-Identifier     |            是 | 根据验证方法给出的用户识别码。例如 - `abhav@arweave.org`                |

### 使用方式

使用此标准的用户可以在 arweave 网络上执行 GraphQL 查询，检索使用 `Vouch-For` 标准且对特定 `address` 进行担保的交易。例如：

```javascript
query {
  transactions(
    tags:{name:"Vouch-For", values:["0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g"]}
  ) {
    edges {
      node {
        id
        tags {
          name
          value
        }
      }
    }
  }
}
```

此查询会返回所有对地址 `0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g` 的担保记录。可以通过额外的过滤条件（例如之前所述由「VouchDAO」实施并指定为验证者的特定持有者）来筛选出被指定为验证者的来源。
