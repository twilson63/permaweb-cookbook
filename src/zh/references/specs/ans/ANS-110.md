# [ANS-110：资产可发现性](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-110.md)

状态：草案

作者：Tom Wilson (tom@hyper.io), Sam Williams (sam@arweave.org), Abhav Kedia (abhav@arweave.org)

## 摘要

本文件指定一种数据协议，允许在永久网（permaweb）上发现并统一显示资产。

## 动机

永久网是一个包含各类数据的丰富集合——媒体、内容、功能与应用。用于识别资产的标准协议可为仪表板、市场与更高层次的永久网服务提供可发现性。通过为交易提供可扩展的数据协议，创作者与发布者可以利用整个永久网的内容可发现性——使其在不同应用与上下文中更容易呈现。

例如，该协议可以启用一个市场或交易所，让用户以友好的方式交易并渲染媒体资产。通过使用这些标识标签扩展资产交易，创作者提供一组清晰且可组合的识别器，市场或交易所可以使用它们来呈现资产的详细描述。另一个例子是搜索引擎服务，可能想要索引特定类型的资产。

## 规范

### 交易格式

为了使用资产可发现性，创作者或发布者可以发送或张贴一个 Arweave 交易，并指定下列标签。

| 标签名称    | _是否选填?_ | 标签值                                                                                                                         |
| ----------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Title       | False       | 最多 150 字符，用于识别内容，该标题可提供资产的快速吸引人描述                                                                  |
| Type\*      | False       | 资产类型。一个或多个：`meme`, `image`, `video`, `podcast`, `blog-post`, `social-post`, `music`, `token`, `web-page`, `profile` |
| Topic\*     | True        | 零到多个主题，可用于基于特定主题定位某类型的资产。例如：一个类型为 meme 的资产可能有以下两个主题：`Funny`, `Sports`。          |
| Description | True        | 最多 300 字符的较长描述，可提供进一步描述资产的详细信息                                                                        |

### 使用方式

这些标签的主要目的是允许内容开发人员利用 GraphQL 找到特定类型或主题的资产交易，以在其应用程序中使用。

```javascript
query {
  transactions(
      first: 100,
        tags: [
          { name: "Type", values: ["meme", "blog-post"] },
          { name: "Topic:Funny", values: ["Funny"] },
          { name: "Topic:Jokes", values: ["Jokes"] }
        ]) {
      edges {
          node {
              id
                owner {
                  address
                }
                tags {
                  name
                    value
                }
        }}
    }
}
```

此 GraphQL 查询根据 `Type` 与 `Topic` 标签过滤，筛选出供汇总列表显示的资产。
