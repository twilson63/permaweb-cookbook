# [ANS-105: 授权标签](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-105.md)

作者：Sam Williams <sam@arweave.org>, Abhav Kedia <abhav@arweave.org>

## 摘要

本标准说明一种机制，可在 permaweb 上任意为任何交易标注其所采用的授权。该标准着重于简单性与易用性，以鼓励在整个 permaweb 生态系统中的广泛采用。

## 动机

Arweave 生态系统的目标是建立一个永久的、集体的共有知识与历史库，任何时间对所有人开放。像原始网络一样，因为其开放性，该生态系鼓励广泛地重用与复制资料。在传统网络上，这导致了大量被广泛分享但缺乏授权信息的内容。

通过简单使用 Arweave 交易与数据条目所提供的标签系统，我们可以避免重蹈覆辙，反而构建一个常态化地传达任一数据项授权的网络。由于 Arweave 标签与其关联数据不可分割，所有遵循本标准的交易之授权信息将被 _以原子方式_ 传达给用户——不可能在不一并传送其授权信息的情况下仅传送该数据的链接。由于 Arweave 标签系统的性质，这可在不对用户体验（例如浏览器中渲染数据时）造成任何强制或显著影响的情况下达成。

## 规范

授权标注格式由 Arweave 网络上任一交易上的一个附加标签，以及两种用于定义授权类型的附加交易格式所组成。

### 将授权指派给交易数据

若要将某笔交易的数据以给定的交易格式发布，仅需加入以下标签：

| 标签名称 | _可选吗?_ | 标签值                           |
| -------- | --------- | -------------------------------- |
| License  | False     | `License Definition` 的 TXID     |
| Title    | True      | 作品的标题                       |
| Creator  | True      | 作品创作者的名称/识别符          |
| Source   | True      | 获取该数据之来源的链接（如适用） |

### 授权声明

若要声明先前上传的交易具有某一授权，请提供以下标签：
| 标签名称 | _可选吗?_ | 标签值 |
|---|---|---|
|App-Name|False|`"License-Assertion"`|
|Original|False|原始上传的 TXID|
|License|False|已更新的 `License Definition` 的 TXID|

### 授权或法律工具定义

若要定义一项授权或相关法律工具（例如 [CC0](https://wiki.creativecommons.org/wiki/CC0)），提交一笔带有下列标签的交易：

| 标签名称     | _可选吗?_ | 标签值                                         |
| ------------ | --------- | ---------------------------------------------- |
| App-Name     | True      | `"License-Definition"`                         |
| Logo         | True      | `License Logo` 的 TXID                         |
| Short-Name   | True      | 用于引用该授权的简短「代称」（最多 64 个字符） |
| Content-Type | True      | 用以描述授权之所包含文件的 MIME 类型           |

交易的 **内容** 必须包含所定义授权的法律文本。

### 授权标志

授权标志交易应如下定义：

| 标签名称     | _可选吗?_ | 标签值                 |
| ------------ | --------- | ---------------------- |
| Content-Type | False     | 所包含文件的 MIME 类型 |

交易的 **内容** 必须包含该授权标志的视觉表示。
