# 交易捆绑/打包

交易捆绑（transaction bundle）是一种特殊的 Arweave 交易类型，可将多个其他交易及/或数据项组合于其中。

由于交易捆绑包含许多嵌套交易，它们是 Arweave 能够扩展到每秒数千笔交易能力的关键。

用户会将交易提交到捆绑服务（例如 [Turbo](https://ardrive.io/turbo-bundler/)），该服务会将它们与其他交易合并为一个「bundle（捆绑）」，并将其发布到网络上。

## 捆绑如何帮助 Arweave？

### 可用性

捆绑服务保证被捆绑的交易能可靠地发布到 Arweave，而不会被丢弃。

被捆绑交易的交易 ID 会立即可用，这表示数据可以立即被访问，就好像它已经在 Arweave 网络上一样。

### 可靠性

发布到 Arweave 的交易有时可能因为多种原因（例如网络活动高峰）而确认失败（导致交易被丢弃）。在这些情况下，交易可能会变成「孤儿」，即卡在 mempool（内存池）中并最终被移除。

捆绑服务通过持续尝试将捆绑数据发布到 Arweave 来解决此问题，确保其不会失败或卡在内存池中。

### 可扩展性

捆绑可存储最多 2<sup>256</sup> 笔交易，这些交易会以单一 Arweave 交易进行结算。这使得 Arweave 的区块空间能够扩展，以支持几乎任何使用场景。

## 什么是嵌套捆绑（Nested Bundles）？

捆绑可以包含要上传到 Arweave 的数据项，这些数据项本身也可以是一个捆绑。

这表示可以上传一个包含多个捆绑的捆绑，换句话说，即为 **嵌套捆绑**。

嵌套捆绑在理论上没有嵌套深度限制，意味着交易吞吐量可以大幅提升。

当你有多个不同分组的捆绑数据，希望能保证它们同时且一并到达 Arweave 时，嵌套捆绑会很有用。
