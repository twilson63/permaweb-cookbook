# 交易捆綁

交易捆綁（transaction bundle）是一種特殊的 Arweave 交易類型，可將多個其他交易及/或資料項目群組於其中。

由於交易捆綁包含許多巢狀交易，它們是 Arweave 能夠擴展到每秒數千筆交易能力的關鍵。

使用者會將交易提交到捆綁服務（例如 [Turbo](https://ardrive.io/turbo-bundler/)），該服務會將它們與其他交易合併為一個「bundle（捆綁）」，並將其發佈到網路上。

## 捆綁如何幫助 Arweave？

### 可用性

捆綁服務保證被捆綁的交易能可靠地發佈到 Arweave，而不會被丟棄。

被捆綁交易的交易 ID 會即時可用，這表示資料可以立即被存取，就好像它已經在 Arweave 網路上一樣。

### 可靠性

發佈到 Arweave 的交易有時可能因為多種原因（例如網路活動高峰）而確認失敗（導致交易被丟棄）。在這些情況下，交易可能會變成「孤兒」，即卡在 mempool（記憶池）中並最終被移除。

捆綁服務透過持續嘗試將捆綁資料發佈到 Arweave 來解決此問題，確保其不會失敗或卡在記憶池中。

### 可擴展性

捆綁可儲存最多 2<sup>256</sup> 筆交易，這些交易會以單一 Arweave 交易進行結算。這使得 Arweave 的區塊空間能夠擴展，以支援幾乎任何使用案例。

## 什麼是巢狀捆綁（Nested Bundles）？

捆綁可以包含要上傳到 Arweave 的資料項目，這些資料項目本身也可以是一個捆綁。

這表示可以上傳一個包含多個捆綁的捆綁，換句話說，即為 **嵌套捆綁**。

嵌套捆綁在理論上沒有巢狀深度限制，意味著交易吞吐量可以大幅提升。

當你有多個不同群組的捆綁資料，希望能保證它們同時且一併到達 Arweave 時，嵌套捆綁會很有用。
