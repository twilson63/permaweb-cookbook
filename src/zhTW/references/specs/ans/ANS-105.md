# [ANS-105: 授權標籤](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-105.md)

作者：Sam Williams <sam@arweave.org>, Abhav Kedia <abhav@arweave.org>

## 摘要

本標準說明一種機制，可在 permaweb 上任意為任何交易標註其所採用的授權。該標準著重於簡單性與易用性，以鼓勵在整個 permaweb 生態系統中的廣泛採用。

## 動機

Arweave 生態系的目標是建立一個永久的、集體的共有知識與歷史庫，任何時間對所有人開放。像原始網路一樣，因為其開放性，該生態系鼓勵廣泛地重用與複製資料。在傳統網路上，這導致了大量被廣泛分享但缺乏授權資訊的內容。

透過簡單使用 Arweave 交易與資料條目所提供的標籤系統，我們可以避免重蹈覆轍，反而建構一個常態化地傳達任一資料項目授權的網路。由於 Arweave 標籤與其關聯資料不可分割，所有遵循本標準的交易之授權資訊將被 _以原子方式_ 傳達給使用者——不可能在不一併傳送其授權資訊的情況下僅傳送該資料的連結。由於 Arweave 標籤系統的性質，這可在不對使用者體驗（例如瀏覽器中渲染資料時）造成任何強制或明顯影響的情況下達成。

## 規範

授權標註格式由 Arweave 網路上任一交易上的一個附加標籤，以及兩種用於定義授權類型的附加交易格式所組成。

### 將授權指派給交易資料

若要將某筆交易的資料以給定的交易格式發佈，僅需加入以下標籤：

| 標籤名稱 | _可選嗎?_ | 標籤值                           |
| -------- | --------- | -------------------------------- |
| License  | False     | `License Definition` 的 TXID     |
| Title    | True      | 作品的標題                       |
| Creator  | True      | 作品創作者的名稱/識別符          |
| Source   | True      | 取得該資料之來源的連結（如適用） |

### 授權聲明

若要聲明先前上傳的交易具有某一授權，請提供以下標籤：
| 標籤名稱 | _可選嗎?_ | 標籤值 |
|---|---|---|
|App-Name|False|`"License-Assertion"`|
|Original|False|原始上傳的 TXID|
|License|False|已更新的 `License Definition` 的 TXID|

### 授權或法律工具定義

若要定義一項授權或相關法律工具（例如 [CC0](https://wiki.creativecommons.org/wiki/CC0)），提交一筆帶有下列標籤的交易：

| 標籤名稱     | _可選嗎?_ | 標籤值                                         |
| ------------ | --------- | ---------------------------------------------- |
| App-Name     | True      | `"License-Definition"`                         |
| Logo         | True      | `License Logo` 的 TXID                         |
| Short-Name   | True      | 用於引用該授權的簡短「代稱」（最多 64 個字元） |
| Content-Type | True      | 用以描述授權之所包含檔案的 MIME 類型           |

交易的 **內容** 必須包含所定義授權的法律文本。

### 授權標誌

授權標誌交易應如下定義：

| 標籤名稱     | _可選嗎?_ | 標籤值                 |
| ------------ | --------- | ---------------------- |
| Content-Type | False     | 所包含檔案的 MIME 類型 |

交易的 **內容** 必須包含該授權標誌的視覺表示。
