# [ANS-105: License Tags](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-105.md)

作者: Sam Williams <sam@arweave.org>, Abhav Kedia <abhav@arweave.org>

## 摘要

本標準概述了一種機制，可在永存網（permaweb）上的任意交易上標註其發布時所採用的授權。該標準強調簡潔與易用性，以鼓勵整個永存網生態系統廣泛採用。

## 動機

Arweave 生態系統的目標是建立一個永久的、共用的知識與歷史寶庫，讓所有人隨時可取得。就像原始的網際網路一樣，其開放性鼓勵資料被廣泛重用與複製。在傳統網路上，這導致大量被廣泛共享的資訊缺乏相關的授權資料。

透過簡單利用 Arweave 交易與資料條目的標籤系統，我們可以避免重蹈覆轍，並建構一個普遍慣例是對任一資料都會傳達其授權資訊的網路。由於 Arweave 標籤與其關聯資料不可分離，所有遵從此標準的交易其授權資料將會被以 _原子性_ 的方式傳達給使用者 ── 無法僅傳遞資料連結而不一併傳送其授權資訊。基於 Arweave 標籤系統的特性，這可在不對使用者體驗（例如瀏覽器中呈現資料時）造成任何強制性或明顯影響的情況下達成。

## 規範

授權標籤格式由在 Arweave 網路上任一交易添加的一個額外標籤構成，此外還有兩種用於定義授權類型的交易格式。

### 為交易資料指派授權

若要在特定交易格式下將交易的資料發布為某授權，只需添加以下標籤：

| 標籤名稱 | _可選?_ | 標籤值                              |
| -------- | ------- | ----------------------------------- |
| License  | False   | TXID of `License Definition`        |
| Title    | True    | 作品標題                            |
| Creator  | True    | 創作者的名稱/識別符（或多位創作者） |
| Source   | True    | 取得該素材的來源連結（如適用）      |

### 授權聲明

若要聲明先前上傳的某筆交易具有特定授權，請提供以下標籤：
| 標籤名稱 | _可選?_ | 標籤值 |
|---|---|---|
|App-Name|False|`"License-Assertion"`|
|Original|False|原始上傳的 TXID|
|License|False|更新後的 `License Definition` 的 TXID|

### 授權或法律工具定義

若要定義一項授權或相關的法律工具（例如 [CC0](https://wiki.creativecommons.org/wiki/CC0)），請提交一筆包含以下標籤的交易：

| 標籤名稱     | _可選?_ | 標籤值                                             |
| ------------ | ------- | -------------------------------------------------- |
| App-Name     | True    | `"License-Definition"`                             |
| Logo         | True    | `License Logo` 的 TXID                             |
| Short-Name   | True    | 一個簡短（最多 64 字元）的「代號」，用以參照該授權 |
| Content-Type | True    | 所含檔案的 MIME 類型，用以描述該授權               |

交易的 **主體** 必須包含所定義授權之法律文本。

### 授權徽標

授權徽標交易應定義如下：

| 標籤名稱     | _可選?_ | 標籤值               |
| ------------ | ------- | -------------------- |
| Content-Type | False   | 所含檔案的 MIME 類型 |

交易的 **主體** 必須包含該授權徽標的視覺呈現。
