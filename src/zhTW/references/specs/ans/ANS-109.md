# [ANS-109：Vouch-For（身份斷言）](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-109.md)

狀態：草案（版本 `0.1`）

作者：Abhav Kedia (abhav@arweave.org), Sam Williams (sam@arweave.org), Tom Wilson (tom@hyper.io)

## 摘要

本文件定義一種交易格式，允許位址在 permaweb 上為其他位址的身份進行擔保。

## 動機

Sybil 抵抗性是 permaweb 上大多數應用所需的必要元件。允許位址為其他位址進行「擔保（vouch）」的交易格式，可使人工與程式化的驗證者（Verifiers）確認位址的真人屬性。其他應用即可將此資訊作為身份驗證的原始構件，並在其上建立各種保護機制。

舉例來說，基於此系統之上的抽象與保護機制之一，可能是「VouchDAO」——一個社群，用以在特定時間點指定他們認為值得信賴的人類「Verifier」或「Verification Services」。

## 規範

### 交易格式

驗證者可以透過送出包含下列標籤的交易，使用 `Vouch-For` 標準來斷言某個 `address` 的身份。

| 標籤名稱            | _是否為選用?_ | 標籤值                                                                  |
| ------------------- | ------------: | ----------------------------------------------------------------------- |
| App-Name            |            否 | `Vouch`                                                                 |
| Vouch-For           |            否 | Arweave `address` 在此交易中被擔保的目標位址                            |
| App-Version         |            是 | `0.1`                                                                   |
| Verification-Method |            是 | 用於驗證該人身份的方法。例如 - `Twitter`/`In-Person`/`Gmail`/`Facebook` |
| User-Identifier     |            是 | 根據驗證方法給出的使用者識別碼。例如 - `abhav@arweave.org`              |

### 使用方式

使用此標準的使用者可以在 arweave 網路上執行 GraphQL 查詢，搜尋使用 `Vouch-For` 標準且對特定 `address` 進行擔保的交易。例如：

```javascript
query {
  transactions(
    tags:{name:"Vouch-For", values:["0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g"]}
  ) {
    edges {
      node {
        id
        tags {
          name
          value
        }
      }
    }
  }
}
```

此查詢會回傳所有對位址 `0L_z90sYv36VDoDhrRBffo9KrADWpCaaGQz7hJhhP9g` 的擔保紀錄。可透過額外的過濾條件（例如先前所述由「VouchDAO」實作並指定為驗證者的特定擁有者）來篩選出被指定為驗證者的來源。
