# [ANS-106：Do-Not-Store Request](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-106.md)

狀態：草稿-1

作者：Abhav Kedia <abhav@arweave.org>, Sam Williams <sam@arweave.org>

## 摘要

本文件描述一種交易格式，Arweave 網路的使用者可用以請求礦工對某些類型的資料不進行儲存（基於各種理由）。為了請求不儲存，需上傳一筆包含相關資料交易 ID 的交易，並附上此處規定的標籤。節點在收到此資料後，將獨立決定是否接受該請求。

## 動機

Arweave 的永久儲存協議實現了對媒體與文件的真正永久數位保存。

可能有各種原因使個人或實體希望將某些資料從網路中移除，包括隱私、政府法規及著作權侵害。透過上傳符合本標準的交易，網路使用者可確保其請求被廣播至所有相關的儲存節點營運者。

## 規範

### 交易格式

Do-Not-Store Request 交易必須是一筆帶有以下標籤的交易：

| 標籤名稱     | 是否可選？ | 標籤值                                                                                                 |
| ------------ | ---------- | ------------------------------------------------------------------------------------------------------ |
| App-Name     | 否         | `Do-Not-Store`                                                                                         |
| Do-Not-Store | 否         | Arweave `Txn ID` of the data in question                                                               |
| Category     | 是         | `Category-Tag`                                                                                         |
| Geography    | 是         | If requesting removal in particular countries, include the 2 letter `Country-Code` (Alpha-2, ISO 3166) |
| Content-Type | 是         | A guide to the type of content included in the body of the transaction, in order to aid rendering.     |

交易的內容（body）必須包含一份文字描述，說明案件理由與為何該資料不應被儲存。可加入多個 Do-Not-Store 標籤以同時請求對多個資料項目不儲存。

### Category-Tags

Category 標籤是指定移除理由的簡短方式。常見標籤包括 `Private`、`Regulation`、`Copyright`。可視情況使用自訂的 category 標籤。自訂標籤不得超過 50 個字元。

## 未來工作

未來可能需要根據資料的各種屬性或與之相關的標籤來請求移除。本標準可擴展以允許交易清單及/或相關的過濾條件。
