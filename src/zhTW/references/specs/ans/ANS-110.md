# [ANS-110：資產可發現性](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-110.md)

狀態：草案

作者：Tom Wilson (tom@hyper.io), Sam Williams (sam@arweave.org), Abhav Kedia (abhav@arweave.org)

## 摘要

本文件指定一種資料協議，允許在永久網（permaweb）上發現並統一顯示資產。

## 動機

永久網是一個包含各類資料的豐富集合——媒體、內容、功能與應用程式。用於識別資產的標準協議可為儀表板、市場與更高層次的永久網服務提供可發現性。透過為交易提供可擴充的資料協議，創作者與發佈者可以利用整個永久網的內容可發現性——使其在不同應用與上下文中更容易出現。

例如，該協議可以啟用一個市場或交易所，讓使用者以友善的方式交易並渲染媒體資產。透過使用這些識別標籤擴展資產交易，創作者提供一組清晰且可組合的識別器，市場或交易所可以使用它們來呈現資產的詳細描述。另一個例子是搜尋引擎服務，可能想要索引特定類型的資產。

## 規範

### 交易格式

為了使用資產可發現性，創作者或發佈者可以發送或張貼一個 Arweave 交易，並指定下列標籤。

| 標籤名稱    | _是否選填?_ | 標籤值                                                                                                                         |
| ----------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Title       | False       | 最多 150 字元，用來識別內容，此標題可提供資產的快速吸睛描述                                                                    |
| Type\*      | False       | 資產類型。一個或多個：`meme`, `image`, `video`, `podcast`, `blog-post`, `social-post`, `music`, `token`, `web-page`, `profile` |
| Topic\*     | True        | 零到多個主題，可用來依特定主題定位某類型的資產。例如：一個類型為 meme 的資產可能有以下兩個主題：`Funny`, `Sports`。            |
| Description | True        | 最多 300 字元的較長描述，可提供進一步描述資產的詳細資訊                                                                        |

### 使用方式

這些標籤的主要目的，是允許內容開發人員利用 GraphQL 找到特定類型或主題的資產交易，以在其應用程式中使用。

```javascript
query {
  transactions(
      first: 100,
        tags: [
          { name: "Type", values: ["meme", "blog-post"] },
          { name: "Topic:Funny", values: ["Funny"] },
          { name: "Topic:Jokes", values: ["Jokes"] }
        ]) {
      edges {
          node {
              id
                owner {
                  address
                }
                tags {
                  name
                    value
                }
        }}
    }
}
```

此 GraphQL 查詢根據 `Type` 與 `Topic` 標籤過濾，篩選出供彙整列表顯示的資產。
