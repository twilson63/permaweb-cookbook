# [ANS-101：閘道能力端點](https://github.com/ArweaveTeam/arweave-standards/blob/master/ans/ANS-101.md)

Status: Draft

Version: -

## 摘要

本文檔描述了一個 HTTP 端點與 schema，Arweave 閘道應暴露該端點，以便閘道的使用者能判斷該閘道所支援的能力。

## 動機

隨著連接到 Arweave 區塊鏈的閘道生態系成長，這些閘道將提供不同的功能。閘道使用者需要能判斷該閘道是否支援他們運行特定應用或為特定用途所需的能力。

## 規範

### 1. HTTP 端點

符合本規範的閘道 MUST 暴露一個 HTTP 端點，該端點在路徑 `/info/capabilities` 回應對 GET 請求，並回傳符合第 2 節 schema 的 JSON 文件，該文件需準確列出閘道所支援的能力。

### 2. 回應結構

由該端點回傳的 JSON 文件 MUST 為一個物件，且包含一個 `capabilities` 鍵，該鍵為一個 `capability` 物件的陣列。

capability 物件至少需具備下列結構 -

- `name` 功能名稱。此欄位 MUST 為全域唯一名稱。
- `version` 功能版本。此欄位 MUST 為 semver 版本字串。

capability 物件 MAY 遵循該功能特定的額外結構。

範例回應主體：（名稱、版本及 capability 物件僅為 _範例_。）

```json
{
  "capabilities": [
    { "name": "arql", "version": "1.0.0" },
    { "name": "graphql", "version": "1.0.0" },
    { "name": "arweave-id-lookup", "verson": "1.0.0" },
    { "name": "post-bundled-tx-json", "version": "1.0.0" },
    { "name": "post-delegated-tx", "version": "1.0.0", "maxFee": "0.00125" },
    {
      "name": "push-on-event-api",
      "version": "1.0.0",
      "platforms": ["push-android", "push-ios", "web-push-firefox", "webhook"]
    }
  ]
}
```
